# 二、核心模块设计（你已有 + 我帮你补齐）

## 1. 项目管理模块（Project）

你现在缺的关键点：

### 必备能力

* Git仓库配置（支持多仓库）

  * HTTPS / SSH
  * 私钥管理（要加密存储）
* 分支策略

  * master / develop / release
* 项目类型识别

  * Java（Maven / Gradle）
  * 前端（Node / Vite / Webpack）
  * 混合项目（前后端分离）

### 建议增强

* **环境隔离**

  * dev / test / prod
* **变量管理（类似CI变量）**

  * DB_URL、Redis、API_KEY

👉 这一步决定你是不是“工具”，还是“平台”

---

## 2. 构建模块（Build / CI）

你目前只是“拉代码 + 打包”，太弱了。

### 必须补充：

#### 1）构建流水线（Pipeline）

不是简单打包，要有阶段：

```
拉代码 → 编译 → 单测 → 打包 → 构建产物
```

#### 2）构建缓存（非常重要）

* Maven依赖缓存
* Node modules缓存

👉 否则你会慢到没人用

---

## 3. 发布模块（Deploy）

你现在设计的是“直接覆盖jar + 重启”，存在风险。

### 必须升级：

#### 1）发布策略（重点）

* 单机发布
* 滚动发布（Rolling）
* 蓝绿发布（Blue-Green）👉 推荐支持
* 灰度发布（进阶）

#### 2）部署方式

* SSH部署（你现在的方式）
* 支持脚本自定义（很关键）

#### 3）启动方式

支持多种：

* `java -jar`
* `systemctl`
* `docker run`

👉 不然扩展性很差

---

## 4. 版本管理（Version Control）

你提到了“版本号自动生成”，但没设计规则。

### 建议标准：

版本号规则：

```
v{日期}-{commitId}-{buildNumber}
例：
v20260416-a1b2c3-001
```

### 必备能力：

* 每次发布生成版本
* 版本对应：

  * Git commit
  * 构建产物
  * 发布记录

---

## 5. 回滚机制（Rollback）

这是你系统的“核心卖点之一”，必须做扎实。

### 必须支持：

* 一键回滚到历史版本
* 回滚方式：

  * 直接替换jar
  * 重新部署历史构建产物

### 关键细节：

* 回滚前自动备份当前版本
* 回滚失败自动恢复

---

## 6. 日志系统（你重点关注的）

你现在只说“监控启动日志”，远远不够。

### 要拆成三类日志：

---

### ① 操作日志（审计日志）

必须有：

| 字段   | 示例               |
| ---- | ---------------- |
| 操作人  | 张三               |
| 操作类型 | 发布 / 回滚          |
| 项目   | xxx-service      |
| 环境   | prod             |
| 版本号  | v20260416-xxx    |
| 时间   | 2026-04-16 21:00 |

👉 用于追责 & 审计

---

### ② 构建日志（CI日志）

* Maven / npm 输出
* 实时流式展示（WebSocket）

👉 必须支持：

* 关键字高亮（ERROR / WARN）
* 快速定位失败步骤

---

### ③ 应用日志（运行日志）

这个你只说“看是否启动成功”，太浅了。

### 必须增强：

* 实时 tail 日志（类似 `tail -f`）
* 支持筛选：

  * ERROR
  * WARN
* 启动检测：

  * 端口检测
  * 健康检查接口（/actuator/health）

👉 建议直接集成类似 ELK Stack 思路（轻量版）

---

## 7. 发布体验优化（重点）

你这个产品成败就在这里。

---

### 1）一键发布（核心功能）

用户只需要：

* 选项目
* 选分支
* 选环境
* 点发布

👉 其他全部自动化

---

### 2）发布前检查（Pre-check）

必须有：

* 服务器是否可连
* 磁盘空间是否够
* 端口是否被占用
* 上一个服务是否正常

👉 否则上线就是赌

---

### 3）发布过程可视化

类似：

```
[✔] 拉代码
[✔] 编译
[✔] 打包
[✔] 上传
[✔] 停服务
[✔] 启动服务
[✔] 健康检查
```

👉 用户安全感来自这里

---

## 8. 权限系统（很多人会忽略）

必须做：

* 发布权限（谁能发 prod）
* 回滚权限
* 日志查看权限

👉 否则上线就是“谁都能搞”

---

## 9. 通知系统（非常实用）

* 发布成功 / 失败通知
* 支持：

  * 企业微信
  * 钉钉
  * 邮件

👉 提升团队协作效率

---

## 10. 失败处理机制（关键）

你现在没提，这是大坑。

### 必须有：

* 发布失败自动停止
* 自动回滚（可选）
* 错误原因提示
