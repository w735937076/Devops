# DRP 发布平台 - 开发任务计划

## 项目概述

**项目名称**: DevOps Release Platform (DRP)
**项目目标**: 面向中小团队的轻量级 DevOps 发布平台
**对标能力**: Jenkins + GitLab CI/CD + Spring Boot Admin

## 任务分组

### 第一批次 (可并行开发)

| 任务 | 说明 |
|-----|------|
| TASK-01 | 项目骨架 + 公共模块，其他任务的前置依赖 |
| TASK-03 | 前端基础框架，独立于后端可并行开发 |

### 第二批次 (依赖第一批次)

| 任务 | 说明 |
|-----|------|
| TASK-02 | 认证服务，依赖 drp-common |
| TASK-04 | 项目管理服务，依赖 auth |
| TASK-05 | 服务器管理服务，依赖 auth |

### 第三批次 (依赖第二批次)

| 任务 | 说明 |
|-----|------|
| TASK-06 | 构建服务，依赖 project、server |
| TASK-07 | 日志服务，依赖 common |
| TASK-08 | 通知服务，依赖 common |

### 第四批次 (核心模块)

| 任务 | 说明 |
|-----|------|
| TASK-09 | 部署服务，依赖 build、project、server、log、notify |
| TASK-10 | 集成测试与部署，依赖所有任务 |

## 技术栈

| 层级 | 技术 |
|-----|------|
| 前端 | Vue 3 + Vite + Element Plus + TypeScript |
| 后端 | Spring Boot 3.2 + Spring Security + Spring Data JPA |
| 数据库 | MySQL 8.0 + Redis 7 |
| 构建 | Maven 3.9 + npm |
| 容器 | Docker + Docker Compose |
| 存储 | MinIO (对象存储) |

## 开发规范

### 代码规范
- 命名: 驼峰命名法
- 包名: com.drp.{模块}
- 分层: Controller → Service → Repository
- 异常: 统一使用 BusinessException

### Git 分支规范
```
main          # 主分支 (生产)
develop       # 开发分支
feature/*     # 功能分支
hotfix/*      # 热修复分支
```

### Commit 规范
```
feat: 新功能
fix: 修复bug
docs: 文档变更
style: 代码格式
refactor: 重构
test: 测试
chore: 构建/工具
```
