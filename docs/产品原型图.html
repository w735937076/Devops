页面说明 - 创建项目

页面目标

通过四步向导完成项目创建，统一采集项目基础信息、Git/凭证配置、构建参数，并在提交前确认生成内容。

向导步骤

Step 1 基本信息 → Step 2 Git 配置 → Step 3 构建配置 → Step 4 完成确认

设计要点

  • 凭证从凭证中心选择，不在项目表单中直接保存敏感信息
  • 构建配置根据项目类型动态变化，并以 JSON 方式存储
  • 默认只采集基础配置，分支策略与环境变量可在详情页继续维护

页面目标

用户操作路径

凭证管理 → 新建凭证 → 填写类型与密钥信息 → 保存加密存储 → 在项目创建/编辑时选择关联

  • 支持 `USERNAME_PASSWORD`、`ACCESS_TOKEN`、`SSH_KEY` 三种凭证类型
