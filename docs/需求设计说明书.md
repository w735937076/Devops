## 1. 产品概述

### 1.1 产品定位

**产品名称**：DevOps Release Platform（DRP）

**核心定位**：面向中小团队（5~50人）的轻量级 DevOps 发布平台，替代人工发版流程，降低上线风险，提升可追溯性。

**对标能力**：简化版 Jenkins + GitLab CI/CD + Spring Boot Admin

### 1.2 目标用户

| 用户角色 | 职责描述 |
|---------|---------|
| 开发人员 | 触发构建、查看日志、申请发布 |
| 运维人员 | 管理服务器、配置部署策略、执行发布 |
| 项目管理员 | 权限管理、项目配置、版本审批 |
| 技术负责人 | 版本回滚、发布监控、审计追溯 |

### 1.3 核心价值

- **自动化**：替代人工打包、上传、重启的繁琐流程
- **安全性**：发布前检查、蓝绿发布、一键回滚
- **可追溯**：完整的操作审计日志、构建记录、版本链路
- **可视化**：实时日志流、发布进度可视化、健康状态监控

### 1.4 专业术语说明

| 术语 | 说明 |
|-----|------|
| Pipeline | 构建流水线，包含拉代码→编译→测试→打包→部署等阶段 |
| Blue-Green | 蓝绿发布策略，保持两套环境，新版本在绿色环境验证后切换流量 |
| Rolling | 滚动发布策略，批量替换实例 |
| Agent | 部署代理，运行在目标服务器的轻量级服务 |
| Build Number | 构建序号，用于版本标识 |

### 3.3 部署策略架构

#### 3.3.1 单机发布

```
[用户触发] → [停止服务] → [备份当前版本] → [部署新版本] → [启动服务] → [健康检查]
```

#### 3.3.2 滚动发布 (Rolling)

```
┌─────────────────────────────────────────────────────┐
│ 原有实例:  [Node1] [Node2] [Node3] [Node4]          │
│                                                     │
│ 第一批:    [NEW ] [Node2] [Node3] [Node4]  (25%)    │
│ 第二批:    [NEW ] [NEW ] [Node3] [Node4]  (50%)    │
│ 第三批:    [NEW ] [NEW ] [NEW ] [Node4]  (75%)     │
│ 第四批:    [NEW ] [NEW ] [NEW ] [NEW ]  (100%)     │
└─────────────────────────────────────────────────────┘
```

#### 3.3.3 蓝绿发布 (Blue-Green)

```
┌─────────────────────────────────────────────────────────────┐
│                          Nginx                              │
│                    (流量切换开关)                            │
└────────────────────────────┬────────────────────────────────┘
                             │
              ┌──────────────┴──────────────┐
              │                             │
              ▼                             ▼
     ┌──────────────┐              ┌──────────────┐
     │  Blue环境    │              │ Green环境    │
     │  (当前版本)  │  ← 切换 →    │  (新版本)    │
     │  v1.0.0     │              │  v1.1.0      │
     └──────────────┘              └──────────────┘
```

#### 4.4.2 流水线配置

```java
public class Pipeline {
    private Long id;
    private Long projectId;
    private String name;
    private List<PipelineStage> stages; // 阶段列表

    // 默认流水线
    // Stage 1: Checkout    - 拉取代码
    // Stage 2: Compile      - 编译代码
    // Stage 3: Test         - 执行单元测试
    // Stage 4: Package      - 打包构建产物
    // Stage 5: Upload       - 上传产物到存储
}

public class PipelineStage {
    private String name;                // 阶段名称
    private List<PipelineStep> steps;  // 步骤列表
    private boolean failFast;          // 失败是否快速停止
}

public class PipelineStep {
    private String name;                // 步骤名称
    private StepType type;             // 步骤类型
    private Map<String, String> params; // 步骤参数
    private int timeout;               // 超时时间(秒)
    private boolean optional;          // 是否可选失败
}
```

#### 4.5.3 发布流程（蓝绿策略）

```
1. [Pre-Check] ──────────────────────────────────────────
   │ 检查服务器连通性
   │ 检查磁盘空间 (>1GB)
   │ 检查端口占用
   │ 检查上一个服务健康状态
   ▼
2. [Backup] ──────────────────────────────────────────────
   │ 备份当前版本到 /backup/{project}/{version}/
   │ 记录备份信息到数据库
   ▼
3. [Deploy to Green] ────────────────────────────────────
   │ 上传构建产物到Green服务器
   │ 执行停服脚本
   │ 替换配置文件
   │ 执行启动脚本
   ▼
4. [Health Check] ────────────────────────────────────────
   │ 等待服务启动 (默认30s超时)
   │ 调用健康检查接口
   │ 检查端口监听
   ▼
5. [Switch Traffic] ──────────────────────────────────────
   │ Nginx切换流量到Green环境
   │ 记录切换时间
   ▼
6. [Verification] ───────────────────────────────────────
   │ 验证新版本服务正常
   │ 发送通知
   ▼
7. [Complete] ────────────────────────────────────────────
   │ 更新部署记录状态
   │ 记录版本映射
```

### 4.6 版本管理模块

#### 4.6.1 版本号规则

```
格式: v{日期}-{commitId}-{buildNumber}
示例: v20260416-a1b2c3d-001

说明:
- v: 前缀
- 日期: 8位日期 (YYYYMMDD)
- commitId: Git commit ID (前7位)
- buildNumber: 当日构建序号 (3位，从001开始)

## 7. 安全设计

### 7.1 认证安全

```
┌─────────────────────────────────────────────────────────────┐
│                    认证流程                                  │
├─────────────────────────────────────────────────────────────┤
│  1. [登录请求] ──────────────────────────────────────────── │
│     │ POST /api/auth/login                                  │
│     │ Body: { username, password }                          │
│     ▼                                                      │
│  2. [验证凭证] ──────────────────────────────────────────── │
│     │ 检查用户名密码                                        │
│     ▼                                                      │
│  3. [生成Token] ─────────────────────────────────────────── │
│     │ JWT Token (15分钟有效期)                              │
│     │ + Refresh Token (7天有效期)                          │
│     ▼                                                      │
│  4. [返回Token] ──────────────────────────────────────────── │
│     │ 前端存储到 localStorage                               │
│     │ 每次请求携带 Authorization: Bearer {token}           │
└─────────────────────────────────────────────────────────────┘
```

### 7.2 敏感数据加密

| 数据类型 | 加密方式 | 存储位置 |
|---------|---------|---------|
| 用户密码 | BCrypt | MySQL |
| SSH密码 | AES-256 | MySQL |
| SSH私钥 | AES-256 | MySQL |
| Git凭证 | AES-256 | MySQL |
| API密钥 | AES-256 | MySQL |
| 环境变量(敏感) | AES-256 | MySQL |
| JWT密钥 | AES-256 | 配置文件 |

### 7.3 发布安全控制

```
┌─────────────────────────────────────────────────────────────┐
│                   发布安全检查清单                           │
├─────────────────────────────────────────────────────────────┤
│  □ 1. 权限检查 ──────────────────────────────────────────── │
│       • 用户是否有该项目的发布权限                           │
│       • 生产环境发布是否需要额外授权                          │
│                                                             │
│  □ 2. 变更确认 ──────────────────────────────────────────── │
│       • 展示即将变更的内容                                   │
│       • 列出将重启的服务                                     │
│       • 预计停机时间                                        │
│                                                             │
│  □ 3. 自动检查 ─────────────────────────────────────────── │
│       • 服务器连通性                                       │
│       • 磁盘空间 (>1GB)                                    │
│       • 端口占用情况                                       │
│       • 上一个版本健康状态                                  │
│                                                             │
│  □ 4. 回滚准备 ─────────────────────────────────────────── │
│       • 自动备份当前版本                                    │
│       • 记录回滚点                                          │
└─────────────────────────────────────────────────────────────┘
