## 1. 任务目标

实现“可审计、可回滚、可编排、可观测”的部署服务，支持：

- 一键部署与部署预览
- 单机/滚动/蓝绿/灰度等多策略
- 生产审批、发布窗口、并发锁控制
- 自动健康检查与失败自动回滚
- 部署日志实时推送与可追溯审计
- 与项目、服务器、构建、通知、日志模块联动

## 4. 关键缺失点补齐（产品 + 架构视角）

### 4.1 当前文档遗漏的关键能力

1. 审批流定义不完整：缺少“谁审批、何时审批、拒绝后处理”的闭环。
2. 发布窗口缺失：生产环境没有时间窗/冻结策略，存在变更风险。
3. 并发控制缺失：未限制同项目同环境并发部署，容易互相覆盖。
4. 灰度策略缺失细粒度参数：无批次比例、间隔、失败阈值、自动推进策略。
5. 与构建版本绑定规则不完整：缺少“版本状态校验、校验码校验”。
6. 与项目分支策略联动不完整：没有结合 `require_approval`、`allow_auto_deploy`。
7. 可观测性不足：缺少阶段级耗时、失败阶段定位、通知记录可视化。
8. 风险分级缺失：没有发布风险评分（环境、策略、变更量、时段）。

### 4.2 补齐后的产品规则

- 生产环境默认必须审批，且发布窗口外禁止执行（紧急发布需二次确认）。
- 高风险策略（蓝绿切流、灰度 >= 50%）触发二次审批。
- 同项目同环境仅允许一条 RUNNING 记录（队列中可排队）。
- 仅允许来自成功构建的 ACTIVE 版本部署。
- 部署失败自动触发：健康检查重试 -> 阈值判断 -> 自动回滚（可配置）。

## 7. 部署流程（优化版）

1. 选择项目、版本、环境、策略，提交部署预览。
2. 执行部署前检查：
   - 项目权限校验（TASK-02 + TASK-04）
   - 分支策略校验（TASK-04）
   - 构建版本校验（TASK-06）
   - 服务器连通性/磁盘/端口校验（TASK-05）
   - 发布窗口校验（新增）
   - 并发锁校验（新增）
3. 计算风险等级并判定是否审批。
4. 通过审批后进入执行器，按策略执行。
5. 执行健康检查与监控指标判定。
6. 失败时根据策略自动回滚或人工介入。
7. 推送通知（TASK-08）并记录日志（TASK-07）。

## 11. 验收标准（更新）

1. 四种部署策略至少各通过 1 条端到端流程。
2. 生产部署在无审批时不可执行。
3. 发布窗口外部署可被阻止并提示。
4. 健康检查失败可自动回滚并记录原因。
5. 部署日志实时展示，阶段进度可追踪。
6. 部署、回滚、审批均有操作日志。
7. 部署事件可触发通知且失败可重试。
8. 同项目同环境并发部署被正确拦截。

## 12. 风险与应对（扩展）

| 风险 | 影响 | 应对 |
|-----|------|------|
| 网络抖动导致远程执行中断 | 部署半完成状态 | 阶段幂等 + 可重试 + 补偿回滚 |
| 版本产物与校验码不一致 | 部署后不可运行 | 部署前强制 checksum 校验 |
| 配置漂移（环境变量/脚本） | 同版本跨环境行为不一致 | 部署前输出配置差异摘要 |
| 并发发布覆盖 | 服务不稳定 | 项目环境级分布式锁 |
| 审批链路故障 | 生产发布阻塞 | 审批降级策略 + 管理员兜底通道 |
